📋 Setup guide — how to get your Google OAuth Client ID

This is a one-time setup that takes about 5 minutes. It's free and you don't need to publish anything.

  1. Go to Google Cloud Console
    Open console.cloud.google.com and sign in with your Google account (the same one you use for Google Docs).
  2. Create a new project
    Click the project dropdown at the top → New Project → name it anything (e.g. "Writing Tracker") → click Create.
  3. Enable the Google Drive API
    In the left menu go to APIs & Services → Library. Search for Google Drive API → click it → click Enable.
  4. Configure the OAuth consent screen
    Go to APIs & Services → OAuth consent screen.
    • Choose External → click Create.
    • Fill in App name (e.g. "Writing Tracker") and your email → click Save and Continue.
    • On the Scopes page, click Save and Continue (no changes needed).
    • On the Test Users page, add your own Google email → Save and Continue.
  5. Create an OAuth Client ID
    Go to APIs & Services → Credentials → click + Create Credentials → OAuth client ID.
    • Application type: Web application
    • Name: anything (e.g. "Writing Tracker Web")
    • Under Authorized JavaScript origins, click Add URI and add:
      http://localhost (if running locally) or your actual domain if hosted online.
    • Click Create.
  6. Copy your Client ID
    A dialog appears with your Client ID — it ends in .apps.googleusercontent.com. Copy it.
  7. Paste it into this file
    Open writing-habit-tracker.html in any text editor (Notepad, TextEdit, VS Code).
    Find the line near the top that says:
    const GOOGLE_CLIENT_ID = '';
    Replace the empty quotes with your Client ID:
    const GOOGLE_CLIENT_ID = '123456789-abc.apps.googleusercontent.com';
    Save the file.
  8. Open the file and sign in
    Double-click the HTML file to open it in your browser. Click Sign in with Google, paste your Google Doc URL, and your heatmap will appear!
💡 Tip — hosting online (optional):
If you want to access this from any device, upload the HTML file to GitHub Pages (free). Then add your GitHub Pages URL as an authorized origin in your Google Cloud Console credential instead of http://localhost.
